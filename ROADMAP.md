# Project Roadmap

This roadmap describes the public, non-sensitive direction of the Edge1 Management Interface. It is intentionally high level and does not expose production credentials, private records, internal addresses, or restricted operational details.

## Current foundation

- Responsive management interface for desktop and mobile use
- Private Library Search with source traceability
- Local read-only API wrappers and fixture fallbacks
- Managed service assets, smoke tests, and operator runbooks
- Staged AI filesystem-change workflow with approval and rollback boundaries
- WW.CX Time Authority observations and longitudinal measurement records
- Public architecture, security, citation, and handoff documentation

## Near-term priorities

### Reliability and validation

- Consolidate common validation commands into a predictable test entry point
- Expand smoke tests for local services and fallback behavior
- Improve error messages and operator-facing health summaries
- Keep deployment and rollback procedures synchronized with implementation

### Interface quality

- Refine mobile navigation and status presentation
- Improve accessibility, keyboard navigation, and screen-reader semantics
- Add clearer source and freshness indicators to search results
- Present service state without exposing sensitive diagnostics

### Documentation and project health

- Maintain public architecture diagrams and component maps
- Keep contributor, security, citation, and release guidance current
- Publish sanitized examples that demonstrate supported workflows
- Track completed work in durable project registers and handoff notes

### Controlled automation

- Preserve staged proposal, inspection, validation, approval, apply, verify, and rollback boundaries
- Expand auditable status and diff capabilities without granting silent production writes
- Improve operator review ergonomics and evidence capture
- Keep restart and rollback actions explicit, bounded, and reversible

## Longer-term direction

- A unified private-first operations console for Edge1 and Big Bird services
- Consistent service discovery and health reporting across approved local components
- Search and retrieval experiences that work well from phones, tablets, and desktops
- Portable, sanitized deployment profiles for repeatable environments
- Stronger archival packaging, provenance, citation, and public project presentation

## Non-goals

This project does not aim to publish private library contents, production databases, secrets, credentials, personal information, unrestricted filesystem access, or unaudited autonomous production changes.

## Status convention

Roadmap items are directional rather than promises of a fixed release date. Completed implementation is documented in the repository history, project registers, and handoff records.
