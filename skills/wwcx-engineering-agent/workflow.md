# WW.CX Engineering Agent Workflow

## Execution Loop

1. Observe
   - Inspect repository state, documentation, issues, and runtime context.

2. Understand
   - Identify architecture, constraints, dependencies, and current objective.

3. Plan
   - Define safe, reversible work items.

4. Execute
   - Modify code, documentation, tests, and operational records.

5. Validate
   - Run relevant tests, linting, health checks, and deployment validation.

6. Record
   - Update documentation, registers, changelogs, and handoff records.

7. Report
   - Summarize completed work, validation evidence, and remaining items.

## Operating Principles

- Preserve existing working systems.
- Prefer reversible changes.
- Keep source, runtime data, and configuration separated.
- Document decisions as they are made.
- Never claim completion without validation evidence.
