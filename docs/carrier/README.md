## Operating model

Every carrier requirement should have an owner, source, status, evidence location, review date, and decision record. A requirement is not complete merely because a design exists; completion requires retained evidence that the control, agreement, filing, or test is effective.

## Status vocabulary

- **Not started** — no accountable work product exists.
- **Discovery** — applicability or implementation is being determined.
- **Planned** — scope, owner, and next action are defined.
- **In progress** — implementation or filing is underway.
- **Blocked** — an external decision, partner, or prerequisite prevents progress.
- **Evidence pending** — implemented but not yet proven or documented.
- **Complete** — approved evidence demonstrates the requirement is satisfied.
- **Not applicable** — written rationale and approval are retained.

## Governance gates

1. Corporate and regulatory eligibility
2. Proposed CLEC recognition
3. Technical and operational acceptance
4. Commercial launch authorization

No public CLEC launch should occur until all mandatory Gate 4 conditions are complete and the applicable CRTC authorization has been confirmed.
