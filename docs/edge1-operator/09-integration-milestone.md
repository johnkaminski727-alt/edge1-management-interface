# Edge1 Operator Integration Milestone

## Purpose

Track the transition from isolated components into a connected operator service.

## Integration targets

- Runtime execution layer
- Protocol adapter layer
- Audit and evidence layer
- Installer and service lifecycle
- MCP workspace connection boundary

## Completion requirements

Before production use:

- protocol requests must map to validated runtime operations;
- all mutations must produce evidence records;
- service startup and restart behavior must be verified;
- authentication material must remain outside source control;
- rollback procedures must be tested.

## Current phase

Integration and validation phase.
