# BigBird Operations Console Architecture

## Domains

The platform is intentionally separated into:

1. Monitoring
2. Analytics
3. Management

## Monitoring

Current capabilities:

- Edge1 API health
- BigBird health
- Messaging health
- Numbering health
- Telephony health
- Time Authority health

## Analytics

Future capabilities:

- historical metrics
- latency trends
- capacity reporting
- incident analysis
- operational reporting

## Management

Future controlled workflows:

Request
|
Preview
|
Validate
|
Approve
|
Execute
|
Audit
|
Rollback

No unrestricted management endpoint exists.
